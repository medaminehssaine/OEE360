Introduction

OEE360 is a comprehensive data analytics platform that allows users to upload datasets, manage models, and perform various types of analysis. The application features both a modern web interface built with Next.js and a powerful Python Flask backend for data processing.

What is OEE?

Overall Equipment Effectiveness (OEE) is a standard metric used in manufacturing to measure productivity. It combines three components: Availability, Performance, and Quality to provide a comprehensive view of equipment efficiency.

Features

Dataset Management

  • Upload CSV and JSON datasets
  • Browse and manage your datasets
  • Real-time dataset preview

Model Management

  • Upload models (JSON, PKL, JOBLIB formats)
  • Track model metadata and performance
  • Use predefined models (SARIMA, LSTM, GRU)

Data Analysis

  • Analyze datasets using selected models
  • Generate forecasts with configurable parameters
  • Visualize analysis results

Real-time Analytics

  • Toggle between static and live data analysis
  • Adjust forecast horizon and lookback window
  • Dynamic parameter configuration

Installation

Prerequisites

Before installing OEE360, ensure you have the following prerequisites installed:

  • Node.js (v16 or higher)
  • Python (v3.8 or higher)
  • npm or yarn package manager

Setup

Follow these steps to set up the OEE360 platform:

1. Clone the repository

git clone https://github.com/medaminehssaine/OEE360.git
cd OEE360

2. Install frontend dependencies

npm install

3. Create Python virtual environment

# Create virtual environment
python -m venv env

# Activate virtual environment (Windows)
env\Scripts\activate.bat
# OR for Mac/Linux:
# source env/bin/activate

4. Install Python packages

pip install -r requirements.txt

5. Generate sample datasets and train models

python scripts/generate_datasets.py
python scripts/train_models.py

Important

Make sure your Python virtual environment is activated before installing packages and running scripts.

Usage

OEE360 provides a comprehensive framework for data analytics and model management.

Dataset Management

To start working with datasets:

# Start the Flask backend
python app.py

# In a separate terminal, start the Next.js development server
npm run dev

Navigate to http://localhost:3000 to access the web interface. You can then:

  • Upload new datasets through the web interface
  • Browse existing datasets
  • Preview dataset contents
  • Delete datasets when no longer needed

Model Management

The platform supports various model formats and provides built-in models:

# Available predefined models:
- SARIMA (Seasonal AutoRegressive Integrated Moving Average)
- LSTM (Long Short-Term Memory)
- GRU (Gated Recurrent Unit)

Data Analysis

To perform analysis on your datasets:

# Example API call for analysis
POST /api/analytics
{
  "dataset_id": "your_dataset_id",
  "model_id": "your_model_id",
  "forecast_horizon": 24,
  "lookback_window": 168
}

API Reference

Detailed API documentation for the main components:

Dataset API

GET /api/datasets

Retrieves a list of all available datasets.

Response: JSON array of dataset objects with metadata.

POST /api/datasets

Uploads a new dataset to the platform.

Request: Multipart form data with dataset file.

Supported formats: CSV, JSON

GET /api/datasets/:id

Retrieves a specific dataset by ID.

Parameters:

  • id - Dataset identifier

DELETE /api/datasets/:id

Deletes a dataset from the platform.

Parameters:

  • id - Dataset identifier

Model API

GET /api/models

Retrieves a list of all available models.

Response: JSON array of model objects with metadata.

POST /api/models

Uploads a new model to the platform.

Supported formats: JSON, PKL, JOBLIB

DELETE /api/models/:id

Deletes a model from the platform.

Parameters:

  • id - Model identifier

Analytics API

POST /api/analytics

Runs analysis with selected dataset and model.

Request body:

  • dataset_id - ID of the dataset to analyze
  • model_id - ID of the model to use
  • forecast_horizon - Number of periods to forecast
  • lookback_window - Historical data window size

Development

Guidelines for extending and contributing to OEE360:

Adding New Features

Backend Features:

  • Add new routes to app.py
  • Create corresponding API endpoints in Next.js API routes
  • Update data models and validation schemas

Frontend Features:

  • Add new pages in the app directory
  • Create reusable components in the components directory
  • Update TypeScript interfaces and types

Running Tests

# Run frontend tests
npm test

# Run backend tests
python -m pytest
